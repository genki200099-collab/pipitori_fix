# ディープデバッグ：ゲーム進行・UI点検

作成日時：2026-07-05 11:00 JST

## 点検対象
- ピック候補選択
- 勝者のピック待ち
- ペア選択
- 切断時の進行復旧
- ピック画面の分かりやすさ
- ルールヘルプの開閉
- CPU進行
- Render設定

## 見つけた進行系リスクと修正

### 1. 勝者がピック選択前に切断すると進行が止まる可能性
前回まで、候補選択者の切断とペア選択者の切断には復旧処理があった。
しかし、勝者が裏向きカードを選ぶ場面で切断した場合は、再送だけが続き、ゲームが止まり続ける可能性があった。

#### 修正
勝者が人間かつ切断中で、ピック可能時刻から12秒経過した場合、サーバー側でランダム位置を1枚ピックして進行を復旧する。

CPUと同じくカード中身は見ない。
ランダム配置上の位置を選ぶだけなので、ピック公平性は維持。

### 2. ピック関連の再送タイマーが次のピックに残る可能性
`lastPickTargetRebroadcastAt` / `lastPairChoiceRebroadcastAt` が次のピックに残ると、初回の案内再送が遅れる可能性があった。

#### 修正
新しい pendingPick 作成時と、ピック終了時にピック関連の再送時刻をリセット。

## 見つけたUI系リスクと修正

### 3. 切断中のピック待ち表示が分かりづらい
ピック候補選択者、ピック勝者、ペア選択者が切断中でも、画面上は通常の「選択待ち」に見える可能性があった。

#### 修正
切断中の場合は以下のように明示表示。
- `○○ が切断中です。自動復旧を待っています。`
- ランダム候補で進行する/スキップ扱いで進行する旨を補足。

## 維持した重要仕様
- 最弱側候補選択画面はソート表示。
- 勝者側ピック画面はランダム配置。
- CPUはピック時にカード中身を見ない。
- 切断復旧時の自動ピックもランダム位置選択で、中身は見ない。
- ババブタ公開ピック演出を維持。
- シュート・ザ・ピッグ演出を維持。
- ？ヘルプモーダルの閉じる修正を維持。
- Render設定を維持。
