# ピピトリ v16 CPUセリフ視認性監査

実施日：2026-07-20 JST

## 対応内容

- サーバーがCPU発言ごとに、原文とは別のスマホ用 `compactText` を生成するようにした。
- スマホ表示では話者名が別欄にあるため、重複する相手名や長い前置きを省き、カード情報と行動内容を優先する。
- 例：
  - `💧スートは失点が重いです。12 💧を処理します。` → `💧12を処理します。`
  - `かももどきさん、ごめん！自由なら大胆にいくぞぉ〜✊🏻` → `大胆にいくぞぉ〜✊🏻`
- スマホ縦は最新2件を維持し、通常高さでは各コメントを最大2行で表示する。
- 高さ650px以下の縦画面は従来の安全領域を維持し、短縮文を1行表示する。
- スマホ横は操作領域を優先して最新1件・短縮文を表示する。
- PCは従来どおり原文を表示する。
- 原文は `title`、`aria-label`、進行ログに保持し、情報を失わない。
