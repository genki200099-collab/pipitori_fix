# 扇形手札・選択ルール・切断復帰 v7 実装監査

作成日：2026-07-18 JST

## 対応内容

### 扇形手札

- 通常プレイの手札をPC、スマホ縦、スマホ横のすべてで扇形へ変更した。
- 最大13枚の中心位置を手札幅の77〜80%へ均等配置し、一画面内へ収める。
- 全カード左上に高コントラストの数字＋スート角を固定した。ババブタも `J`＋`🃏` で識別できる。
- スマホ縦430px以下は端の回転を±7.5度へ抑え、回転中も数字・スート角を切らさない。
- 初期配札と新たに加わったカードだけ `fan-enter` で展開する。手番更新では全手札を再アニメーションしない。
- 3枚パスと開始時ペア捨てはカード全体を比較する必要があるため、従来の横スクロールを維持した。

### カード選択

- 1回目のタップで選択し、上へ持ち上げる。
- 選択カードは回転を0度へ戻し、発光、`aria-pressed="true"`、「もう一度」を表示する。
- 同じカードの2回目のタップでのみ `play` を送る。別カードなら選択を移動する。
- `pointerup` 後の合成 `click` だけを除外し、素早い2回タップ自体は妨げない。
- 最後尾の `:last-child` が選択スタイルに勝つCSS詳細度競合を修正した。
- カード確定と同時に切断した場合は送信中フラグを解除し、復帰後に再操作できる。

### レスポンシブ・吹き出し・演出

- 手札ドック内の案内を上端へ固定し、扇形カードの高さを削らないようにした。
- マッド注意の大きな通知は通常手札中に重ねず、カード自身の危険表示を正とした。
- CPU吹き出しと進行文は既存の卓下端レールを維持し、長文を行数制限した。
- 選択リフト、確定パルス、カード追加時の時間差展開、再接続スピナーを追加した。
- `prefers-reduced-motion` では新しい動きを停止する。

### 対戦中ルール表示

- 静的な「全選択肢の説明」を非表示にした。
- 現在の対戦状態から、ラウンド数、手札失点、ババブタ、マッド、シュート、ピック候補数、3枚パス、開始ペアの8項目を生成する。
- 共通進行と選択済み設定だけを表示し、未選択設定を混在させない。

### 切断復帰

- プレイヤーごとに48桁の暗号学的乱数による復帰トークンを発行する。
- 部屋コード＋プレイヤーID＋復帰トークン一致で、既存プレイヤーオブジェクトの同じ席へ戻る。
- 誤トークンと同名だけの席取得を拒否する。旧形式のトークンなし席だけ後方互換を残す。
- WebSocketへ10秒間隔の `ping` / `pong` と35秒の無応答検知を追加した。
- 切断、再読み込み、`online`、`pageshow`、タブ再表示から自動復帰する。
- 即時〜最大10秒のバックオフ、8秒の復帰ハンドシェイク監視、手動再試行ボタンを追加した。
- 別タブが復帰した時は古い接続をコード4000で閉じ、再接続競合を止める。
- 3枚パス中の切断は30秒後にランダム3枚、開始ペア中は30秒後にスキップし、セットアップ停止を防ぐ。

## 実ブラウザ検証

| ビューポート | モード | 13枚の角間隔 | 選択リフト | 結果 |
|---|---|---:|---:|---|
| 320×568 | スマホ縦 | 22.52px | 16.04px | 合格 |
| 844×390 | スマホ横 | 57.12px | 10.56px | 合格 |
| 1366×768 | PC | 69.65px | 20.22px | 合格 |

全ビューポートで次を実測した。

- 13枚すべての数字・スート角が画面内にある。
- 各角中心の `elementFromPoint` が該当カードへ命中し、重なり越しでもタップできる。
- 最初と最後のカードを選択して回転を解除しても、カード全体が画面内にある。
- 1回目のタップでは手札枚数が変化せず、選択・リフトだけが発生する。
- 2回目のタップで手札が1枚減り、サーバーがプレイを受理する。
- CPU吹き出し、進行文、手札が交差しない。
- ページの縦横あふれがない。
- 選択ルール8項目が表示され、旧静的ルール一覧は非表示。
- WebSocketを強制切断後、自動で同じプレイヤーID・復帰トークン・手札へ戻る。

静的寸法検証は縦320×568 / 360×640 / 390×664 / 393×852 / 430×800、横667×375 / 844×390 / 932×430を通過した。

## 自動テスト

- `npm test`：合格。
  - 標準ルール、失点4方式、マッド非重複、シュート条件、UIコントラスト、扇形寸法契約。
- `npm run test:reconnect`：合格（2.862秒）。
  - 誤トークン拒否。
  - 同じプレイヤーIDへ復帰。
  - 13枚の同一手札を復元。
  - 復帰直後のカードプレイを受理。
- `npm run test:smoke`：合格（108.269秒）。
  - `lobby → playing → finished`。
  - 人間プレイ9回、候補選択3回、全4スート、主要CPUコメント11種を観測。
  - 最終得点は全員数値で確定。
- `node --check server.js`：合格。
- インラインJavaScriptのAcorn構文解析：合格。

## 維持した不変条件

- 勝者、最弱、マストフォロー、得点、ピック公平性は変更していない。
- CPUは裏向きカードの内容を見ない。
- マッド・ピッグへ通常の💧失点を重複加算しない。
- 💧の内部IDは `mud` のまま。
- ゲーム開始後はロビーを表示しない。

## インフラ上の境界

通常の回線断、タブ再読み込み、ブラウザのタブ破棄は同じサーバープロセス内で自動復帰できる。現行の部屋状態はメモリ上にあるため、Renderプロセス自体の再起動をまたぐ復旧にはRedisやデータベースなどの外部永続化が必要である。
