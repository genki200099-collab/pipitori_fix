# ピピトリ v26 CPUコメント変数展開・中央セリフ時間調整監査

## 報告された症状

通常CPU実況に次のような文字列が表示されることがあった。

- `[object Object]で主役を取りにいく！`
- `[object Object]を優先処理します。`

また、トリック後に表示する画像付き中央セリフを0.2秒長く表示する要望があった。

## 原因

`cpuPersonaLineFor()`では、カードを一度`cardText()`で表示文字列へ変換していたが、その後の`Object.assign(..., ctx)`によって、元のカードオブジェクトが`card`／`drawn`へ再度上書きされていた。

テンプレート展開側では値を`String(value)`で文字列化していたため、JavaScriptオブジェクトの既定表現である`[object Object]`が画面に出た。

## 修正

### server.js

- `personaContextText()`を追加。
- `cpuPersonaLineFor()`の最終段で、全テンプレート変数を表示用文字列へ正規化。
- `card`と`drawn`はカードオブジェクトの場合に必ず`cardText()`を通す。
- 名前系の値にオブジェクトが来た場合も`name`、`text`、`label`を安全に採用。

### cpu_personality_dialogue.js

- `safeTemplateValue()`を追加。
- サーバー側で正規化漏れが起きても、オブジェクトを直接`String()`しない防御を追加。
- カードオブジェクトはランクとスートアイコン、ババブタは専用名称へ変換。

### 中央セリフ表示時間

各画像付き中央セリフを従来より200ms延長した。

- 通常：2100ms → 2300ms
- ペア浄化：1100ms → 1300ms
- マッド・ピッグ：1300ms → 1500ms
- ババブタ：1350ms → 1550ms

次トリック開始前に退場を完了する契約を維持するため、対応するピック結果確認時間も200ms延長した。

- 通常：2600ms → 2800ms
- ペア浄化：3400ms → 3600ms
- マッド・ピッグ：4200ms → 4400ms
- ババブタ：5000ms → 5200ms
