# ゲーム全体ソースコード点検・UI改善

作成日時：2026-06-28 18:46 JST

## 点検対象
- ババブタ失点デフォルト -20
- ピック時のランダム表示
- CPUのピック候補選択
- 上部ロゴ差し替え
- 作成日時の日本時間表記
- 順番表示の簡素化
- iPhone / Android / PC レスポンシブUI
- 画面のちらつき

## 修正内容

### 1. ルール表示の矛盾修正
部屋作成の初期値は -20 になっていたが、ルールサマリ文に「初期値-50点」が残っていたため、-20点に統一。

### 2. レスポンシブ判定の一本化
`applyResponsiveMode()` と `applyDeviceUiMode()` が両方 `mobile-table-compact` を触っていたため、
`applyResponsiveMode()` は `applyDeviceUiMode()` を呼ぶだけにして、端末判定の矛盾を解消。

### 3. 通常プレイ中の卓面ちらつき軽減
`renderTable()` に通常プレイ中の再描画キャッシュ `__lastTableRenderKey` を追加。
同じ状態の再送信では卓面DOMを作り直さず、カード・順番表示・矢印バッジのちらつきを軽減。

### 4. ヘッダー固定表示の見直し
ロゴ差し替え後にヘッダーが大きくなったため、sticky固定を解除。
スマホ・PCともゲーム盤面を圧迫しにくくした。

### 5. 順番表示の視認性と安定化
下段の順番表示は維持しつつ、タッチ端末では過剰な点滅/アニメーションを抑制。
現在手番は色と枠で分かるように調整。
