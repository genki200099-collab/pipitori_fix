# ルール個別ヘルプ：×ボタンで閉じられないバグ修正

作成日時：2026-07-05 09:24 JST

## 原因
前回、`？` ボタンのスマホ操作対策として `pointerup` / `click` / `touchstart` を強化したが、閉じる側の `×` ボタンと背景は主に `click` だけに依存していた。

端末・ブラウザによっては、`touchstart.preventDefault()` や pointer/click の発火順により、閉じる操作が取りこぼされる可能性があった。

## 修正内容
- `closeRuleHelp(ev)` をイベント引数対応に変更。
- `×` ボタンと背景クリックを `pointerup` / `click` の両方で処理。
- `touchstart` / `pointerdown` で背後の要素へイベントが抜けないようにした。
- 120ms以内の二重発火を抑制。
- Escキーの閉じる処理もイベント安全化。
- 将来モーダル内HTMLが再生成されても閉じられるよう、document委任でも `[data-rule-help-close]` を拾う。
- `×` ボタンに `z-index` と `touch-action: manipulation` を追加。

## 類似バグ点検
- `？` ボタンの開く処理：pointerup/click/キーボード対応維持。
- 背景クリック：pointerup/click対応。
- Escキー：二重登録防止維持。
- ヘルプモーダル以外のオーバーレイ演出は自動消滅式で、閉じるボタン依存なし。
- ピック/ペア選択などのゲーム進行ボタンには影響なし。
