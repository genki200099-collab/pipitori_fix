## 見つけた重要バグと修正

### 1. CPUコメント処理の未定義関数参照
CPUセリフ強化版で、サーバー側の `cpuNextName()` が存在しない `nextPid()` を参照していた。

#### 影響
CPUがカードを出す時やコメントを生成する時に、状況によってサーバー側で `ReferenceError: nextPid is not defined` が発生し、ゲーム進行が止まる可能性があった。

#### 修正
`cpuNextName(room, pid)` をサーバー側で完結する計算に変更。

```js
const np = (Number(pid) + 1 + room.players.length) % room.players.length;
```

### 2. シュート・ザ・ピッグのラウンド毎設定と最終得点の整合性
仕様上、ババブタ失点タイミングが「ラウンド毎」の場合、シュート・ザ・ピッグは毎ラウンド適用される。
しかし実装上、ラウンド途中で発動したシュートのマッド・ピッグ免除が、最終得点側へ十分に反映されない可能性があった。

#### 修正
- `jokerPenaltyTiming === 'perRound'` の場合、シュート発動時点で最終得点側のマッド・ピッグ失点免除フラグも立てる。
- `gameEnd` の場合は従来通り、最終ラウンド発動時のみ免除。

### 3. 古いコメント文言の整理
サーバーコメントに「第2ラウンド補充時」という古い文言が残っていた。
現在は1〜6ラウンド可変なので「次ラウンド補充時」に修正。

### 4. 旧UIバッジ/矢印CSSの安全化
現在の卓面は `seat-status-row` が正。
古い `seat-arrow` / `seat-flow-chip` / `play-badge` / `turn-flag` は描画しない方針。
旧CSSが残っていても表示されないよう、最終上書きCSSを追加。

## 確認した重要仕様

### ピック公平性
- 最弱側の候補選択画面はスート順→数字順。
- 勝者側のピック画面はサーバー側でランダム配置。
- CPUもピック時にカードの中身を見ず、一様ランダムに位置を選ぶ。
- ピック結果は公開情報。

### ラウンド終了
- 手札0枚は終了候補。
- ババブタ1枚だけは、その人の手番開始時に終了。
- 通常カード＋ババブタから通常カードを出した場合、トリック解決とピックまでは進む。

### 得点
- `flat3`: 通常手札1枚-3点。
- `faceValue`: 数字分失点。マッドON時の手札♠11は-40点。
- `spadeSuit`: 1枚-1点、♠は-3点。マッドON時の♠11は♠失点ではなくマッド失点を優先。
- ババブタ失点はデフォルト-20。
- シュート・ザ・ピッグの他者-10は二重加算しない。

### UI

- WIN/PICK/NOW は座席内ステータス行に表示。
- ババブタ公開ピック演出あり。
- シュート・ザ・ピッグ演出あり。
